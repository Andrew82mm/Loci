## Phase 1 — Стабилизация MVP *(в работе)*

Починить критические баги и сделать систему надёжной в ежедневном использовании.

- **Pydantic-схемы** — типизированные модели для всех сущностей (`Message`, `Fact`, `Entity`, `Snapshot`)
- **Фикс потери данных** — сейчас буфер очищается даже если суммаризация упала. Сначала архив, потом очистка
- **Sliding window** — после суммаризации сохранять последние 4 сообщения, а не обнулять буфер полностью
- **Token-based trigger** — суммаризация по количеству токенов, а не сообщений (одно сообщение на 20K токенов = проблема)
- **Chroma в снэпшоты** — сейчас rollback восстанавливает markdown, но векторная БД остаётся в будущем → рассинхрон
- **Score threshold в RAG** — сейчас возвращаются 5 результатов всегда, включая нерелевантные
- **Markdown-aware chunking** — разбивка по заголовкам вместо фиксированных 800 символов
- **SQLite граф** — relations хранятся в sqlite вместо regex-парсинга markdown при каждом запросе
- **Мультиязычный эмбеддинг** — заменить английскую модель на `paraphrase-multilingual-MiniLM-L12-v2`
- **Фикс циклического импорта** — `rag` → `graph` через dependency injection

## Phase 2 — Production-ready

Надёжность, масштабируемость и расширенные возможности.

- **WAL (Write-Ahead Log)** — защита от рассинхрона при падении процесса
- **Git-backed снэпшоты** — снэпшоты через git вместо full-copy директорий (экономия диска)
- **Entity resolution** — `[[Андрей]]` и `[[Andrey]]` — один узел, не два файла
- **Structured extraction** — JSON mode для извлечения фактов вместо regex по тексту
- **LLM-as-judge** — валидация извлечённых фактов второй моделью
- **Стратегия памяти: threshold vs tool-use** — либо суммаризация по порогу (текущее), либо модель сама решает что запомнить через tool calling (подход MemGPT)
- **Per-namespace Chroma** — отдельная векторная коллекция на каждый проект, переключение за O(1)
- **Token budget в промпте** — умная сборка контекста: pinned → task → summary → RAG hits, всё в лимит
- **BM25 + rerank** — гибридный поиск (векторный + keyword), cross-encoder для финального ранжирования
- **File locking** — защита от одновременного запуска двух процессов
- **CLI на typer** — заменить if-elif каскад на нормальные команды с автодополнением

## Phase 3 — Продвинутые возможности

Конкурентные преимущества, требующие существенной работы.

- **Episodic vs semantic memory** — разделить "что произошло" и "что верно сейчас"
- **Temporal knowledge graph** — факты с временными метками: "где работал X в 2023"
- **Multi-hop retrieval** — GraphRAG-style: планировщик разбивает запрос на подзапросы, обходит граф
- **Forgetting curve** — факты с низким score архивируются автоматически
- **MCP server** — подключение Claude Code и других агентов напрямую к памяти
- **Obsidian plugin** — side-panel с RAG-hits для текущего текста
- **Web UI** — визуализация графа (cytoscape.js), FastAPI бекенд, Docker
- **Local LLM (Ollama)** — работа полностью офлайн без OpenRouter
- **Evaluation harness** — автоматические бенчмарки (LongMemEval, LoCoMo), CI блокирует регрессию
